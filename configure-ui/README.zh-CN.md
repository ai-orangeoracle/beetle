# Beetle 配置前端

[English](README.md)

## 这是什么？

**Beetle Configure UI** 是 **Beetle（甲壳虫）** 固件配套的配置 Web 应用。它是一个独立前端，通过 HTTP 与设备通信：在浏览器中打开，连接到设备（通过设备热点或同一局域网），即可配置 WiFi、LLM、各通道等。该 UI **不**打包进固件，而是由设备内置一份供本地访问，或使用在线构建（如 GitHub Pages）打开，从而固件体积更小，配置体验也可单独更新而无需刷机。

## 为什么要有这个库？

- **固件瘦身**：不在固件里塞入整份配置页前端；设备可只带一份精简版或由用户使用在线版。
- **一份代码两处用**：同一套 React 应用既可在「从设备打开」时用，也可在「从网页打开」时用，功能与中英文一致。
- **迭代方便**：配置页可独立改版、发布（如 GitHub Pages），无需重新编译或 OTA 固件。

## 是做什么用的？

连接 Beetle 设备后，用本 UI 可以：

- 设置或修改 **配对码**（保存配置等写操作前必须设置）。
- 配置 **WiFi**（扫描、选择并连接）。
- 配置 **通道**：Telegram、飞书、钉钉、企微、QQ 频道、Webhook（Token、密钥、开关等）。
- 配置 **LLM**：API Key、模型、提供商、兼容接口 URL（如 Ollama）。
- 配置 **代理**、**搜索 Key** 等。
- 查看 **系统信息**、**重启**、**OTA**（若固件启用）、**恢复出厂**。

所有写操作都需正确配对码；页面会代为携带。

## 普通用户：如何使用

### 前置条件（使用配置页之前必须满足）

以下两点**缺一不可**：

1. **已烧录甲壳虫（Beetle）固件的设备。**  
   配置页只能与运行 **Beetle（甲壳虫）固件** 的设备通信。若尚未烧录固件，请先按**上级仓库的 README 或文档**完成固件构建与烧录。本配置页**不能替代**烧录好的设备。

2. **设备已上电且你的手机/电脑能访问到设备。**  
   - **首次使用 / 设备尚未连你家 WiFi 时**：设备会开启 **WiFi 热点**，SSID 为 **Beetle**（无密码）。你的手机或电脑需**先连接这个热点**，然后打开 **http://192.168.4.1**（与固件 SoftAP 地址一致）。
   - **设备已配网并连上家里路由器后**：你的手机或电脑需与设备在**同一局域网**，使用路由器分配给设备的 IP。

**重要**：无论你是通过「设备地址」打开配置页，还是通过「GitHub 页面地址」打开，**浏览器所在设备必须与甲壳虫设备在同一网络**（连接设备热点 Beetle，或与设备在同一局域网）。否则页面无法与设备通信。

---

### 方式一：从设备直接打开配置页

在浏览器里输入设备地址，由设备提供或跳转到配置页。

**设备尚未连你家 WiFi 时（首次使用）：**

1. 设备上电 → 会开启热点 **Beetle**（无密码）。
2. 手机或电脑 **连接到 WiFi「Beetle」**。
3. 浏览器打开 **http://192.168.4.1**（端口 80，固件 SoftAP 地址）。

此时只有设备在该热点下；固件 SoftAP 地址为 192.168.4.1。

**设备已连上你家 WiFi 时：**

- 在与设备**同一局域网**的任意设备上，使用路由器分配给设备的 IP。

**首次打开配置页时**：在页面上设置 **6 位配对码**。配对码用于保护所有写操作；密钥仅保存在设备端。忘记配对码可在配置页使用 **恢复出厂**（需仍能打开配置页）。

---

### 方式二：使用 Git（GitHub）页面地址打开配置页

从互联网打开**同一套**配置页（例如 **https://ai-orangeoracle.github.io/beetle/**）。若要**真正配置设备**，仍然需要已烧录固件的设备，并且浏览器必须能访问到设备（与上面「同一网络」要求一致）。**在线页面不会替代烧录固件或连接设备网络。**

**使用 Git（GitHub Pages）地址时的完整步骤：**

1. **准备好设备**  
   - 在硬件上烧录好甲壳虫固件（若未烧录，请先参考上级仓库文档完成构建与烧录）。  
   - 设备上电。

2. **让手机或电脑与设备处于同一网络**  
   - **未配网时**：手机/电脑连接设备的 **Beetle** 热点（无密码）。  
   - **已配网时**：确保手机/电脑与设备在同一局域网（如同一台家用/办公室路由器）。

3. **打开在线配置页**  
   - 浏览器访问：**https://ai-orangeoracle.github.io/beetle/**  
   - （若仓库配置了自定义域名，则以该域名访问。）

4. **在页面中填写设备地址**  
   - 在配置页找到 **「设备地址」**（Device URL）输入框。  
   - 连接设备热点时填 **http://192.168.4.1**（固件 SoftAP 地址）；同局域网时填路由器分配给设备的 IP。
   - 保存后，页面会向该地址请求设备 API。

5. **设置配对码并开始配置**  
   - 首次使用需在配置页设置 **6 位配对码**。  
   - 之后即可配置 WiFi、通道、LLM 等；所有写操作都会携带该配对码（由页面自动携带）。

**若页面提示无法连接设备**：请检查 (1) 设备已上电；(2) 你已连接 **Beetle** 热点或与设备在**同一局域网**；(3) 设备地址是否正确——连接热点时用 **http://192.168.4.1**，同局域网时用设备在局域网中的 IP。

**仅用在线地址预览界面**：可以在没有设备的情况下打开 GitHub Pages 地址查看界面；若要真正读取或修改配置，必须有一台已烧录固件的设备，并且你的浏览器所在设备与它在同一网络（如上所述）。

---

### 首次使用与配对码

- **首次访问**：在配置页设置 **6 位配对码**。用于保护保存配置、重启、OTA、恢复出厂等写操作；密钥仅保存在设备端。
- **忘记配对码**：在配置页使用 **恢复出厂**（需仍能打开配置页并执行该操作）。

更多说明（配置键、健康接口、配网步骤）：见上级仓库 **docs** 目录（如 `docs/zh-cn/configuration.md`）。

## 开发者：如何使用

### 环境要求

- Node.js 20+  
- npm

### 命令

| 命令             | 说明               |
|------------------|--------------------|
| `npm ci`         | 安装依赖           |
| `npm run dev`    | 启动开发服务       |
| `npm run build`  | TypeScript + Vite 构建 |
| `npm run lint`   | 运行 ESLint        |
| `npm run preview`| 预览生产构建       |

### 本地开发

```bash
cd configure-ui
npm ci
npm run dev
```

开发时 base 为 `/`；页面会向当前访问的 host 请求设备 API（例如连上设备热点后在同一台电脑浏览器打开，即对该设备操作）。生产构建（如 GitHub Pages）会使用 `VITE_BASE_PATH=/<repo>/`，保证资源路径正确。

### 设计与样式

- 界面与样式约定（Token 化、布局、禁止硬编码色值等）见 **docs/DESIGN.md**。
- 新增或修改 UI 时请遵守其中的设计约束。

### 发布

推送到 `main` 且变更涉及 `configure-ui/**` 或 Pages 工作流时，会自动构建并发布到 GitHub Pages。一次性配置：在仓库 **Settings → Pages** 中将 **Source** 选为 **GitHub Actions**。若使用自定义域名，在 **Settings → Pages → Custom domain** 中填写域名，并在 DNS 处添加相应记录（子域名 CNAME 指到 `<owner>.github.io`，根域名用 A 记录）。
