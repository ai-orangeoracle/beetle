# Agent 工具说明

**中文** | [English](../en-us/tools.md)

设备上的 AI Agent 在与你对话时可以使用以下**工具**。你不需要手动调用——Agent 会根据你的话自动选择是否使用、使用哪个工具。

---

## 工具一览

| 工具 | 作用 | Agent 可能用到的场景 |
|------|------|------------------------|
| **get_time** | 返回当前 UTC 时间（日期、星期、时分秒）。 | 你问「几点了」「今天几号」「什么时候执行」等。 |
| **cron** | 解析 5 字段 cron 表达式，返回下次触发时间（UTC）。 | 你问「这个 cron 下次什么时候跑」「下次触发时间」等。 |
| **files** | 列出或读取设备存储（SPIFFS）中的文件；路径在存储根下，禁止 `..`。 | 你要求「列出某目录」「读某个文件」（如 skills、config、memory）。 |
| **web_search** | 按关键词搜索网络并返回简短摘要。 | 你问近期信息、事实，或「搜一下 …」。 |
| **analyze_image** | 根据图片 URL 用视觉模型分析图片内容。 | 你发图片链接并问「这是什么」「描述一下这张图」。 |
| **fetch_url** | 用 HTTP GET 拉取 URL，返回响应正文（文本，会截断）。仅支持 http(s)。 | 你要求「打开这个链接」「把这个网址的内容取回来」。 |
| **http_post** | 向指定 URL 发 HTTP POST，带请求体，返回响应。仅支持 http(s)。 | 你要求触发 webhook、调用 API（如 Home Assistant、IFTTT、n8n）或往某处推送数据。 |
| **remind_at** | 设置定时提醒；到点会给你发一条带提醒内容的消息。 | 你说「X 点提醒我」「在 X 时间提醒我 …」。 |
| **kv_store** | 持久化键值存储（重启不丢）。操作：get / set / delete / list_keys。key 仅允许字母数字、`_`、`-`、`.`，最长 64 字符；value 最长 512 字节；总条目上限 64。 | 你要求「记住 X」「保存我的偏好」「我之前设的 … 是什么」「列出你存了哪些 key」。 |
| **update_session_summary** | 把当前对话写成简短摘要，供后续对话参考。 | Agent 在长对话的自然断点处调用，便于之后回顾。 |
| **board_info** | 返回设备状态：芯片型号、堆/PSRAM 空闲、运行时间、IDF 版本、资源压力、WiFi 是否连接、SPIFFS 存储（总/已用/剩余）。 | 你问「设备状态」「内存还剩多少」「连上 WiFi 了吗」「存储空间」「什么芯片」等。 |

**可选（需开启 feature `gpio`）：**

| 工具 | 作用 | Agent 可能用到的场景 |
|------|------|------------------------|
| **gpio_read** | 读取 GPIO 引脚电平（0 或 1）。仅允许引脚 2、13。 | 你问「读一下 X 引脚」「引脚是不是高电平」。 |
| **gpio_write** | 设置 GPIO 引脚输出电平（0 或 1）。仅允许引脚 2、13。 | 你要求「把 X 引脚拉高/拉低」「打开/关闭 X 引脚」。 |

---

## 限制与说明

- **时间**：设备上的时间需在 NTP/RTC 同步后才准确，可用 **get_time** 查看当前时间。
- **存储（files）**：files 工具为只读；路径不能超出存储根目录。列表最多返回 256 项，读取内容会截断。
- **提醒**：提醒存在设备上，到设定时间会在同一会话/通道给你发一条消息；条数有上限。
- **网络/HTTP**：在内存紧张时，Agent 可能少用或延后使用网络类工具（web_search、fetch_url、http_post、analyze_image），以保持设备稳定。
- **board_info**：用于查看内存、压力、WiFi、SPIFFS 等是否正常，便于你判断设备是否健康或负载高。

若某次工具调用失败（如网络错误、参数不合法），Agent 会用自然语言告诉你结果。
